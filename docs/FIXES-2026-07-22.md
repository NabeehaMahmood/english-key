# Fix log — 2026-07-22 (branch `fix/schema-sql-merge-duplicates`)

The site would not load at all after the latest `main` merge: several PHP files
had syntax errors and `sql/schema.sql` could not be imported. This branch fixes
every blocker and gets all public pages rendering. Each error is documented
below: **what it was, how it was found, who introduced it, how it was fixed, and
its effect.**

Verified locally on Linux (Fedora + system MariaDB) by importing the schema,
running the notes migrations, and rendering every public page through the PHP
CLI — all 9 pages return with `fatals=0`.

## Not fixed on purpose — `sql/migrations/2026-07-20-notes-nav-restructure.sql` is OBSOLETE

`07-20` belongs to an **abandoned lineage** of the notes schema (the "separate
`note_classes` + `note_resource_groups` tables" design). It does
`ALTER TABLE note_resource_groups …`, but migrations `19b`/`19c` intentionally
**drop** `note_resource_groups` and `note_sections` and collapse everything into
one flat `note_classes` table. Applying `07-20` fails with
`Table 'academy.note_resource_groups' doesn't exist`.

The current code confirms the flat lineage is final — `notes.php`,
`admin/notes.php`, and `includes/functions.php` reference **none** of
`note_resource_groups`, `note_sections`, `note_extra_resources`, `nav_style`,
`has_subjects`, or `section_type`.

**➡ Do NOT run `2026-07-20-notes-nav-restructure.sql`.** It should be deleted by
its author (talhaazhar-ta) once confirmed, but is left in place here to avoid
touching unrelated history.
